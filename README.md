# Kakeibo2030（ペイヤン）

ブラウザだけで動く家計簿アプリです。固定費と日々の支出を記録し、月次集計と簡易分析（累計推移・支払方法比率）を確認できます。Google Drive 同期にも対応しています。

## 1. 現在の機能

- 固定費の登録 / 削除
- 支出の登録 / 削除 / 編集（一覧の行クリックで編集）
- 月切り替え表示（前月 / 翌月）
- 月次合計表示（固定費 + 選択月の流動費）
- 分析画面
  - 当月「累計支出」ライン
  - 過去月同日の「累計平均」ライン
  - 支払方法の円グラフ
- カテゴリ・支払方法の追加（入力画面からカスタム追加可能）
- `localStorage` への保存
- Google Drive への手動ログイン同期

## 3. データ構造と保存先

### 3.1 localStorage キー

- `kakeibo_fixed_costs`: 固定費配列
- `kakeibo_expenses`: 支出配列
- `kakeibo_categories`: 追加カテゴリ配列
- `kakeibo_payment_methods`: 追加支払方法配列
- `kakeibo_google_token`: Google OAuth トークン情報
- `kakeibo_app_version`: アプリバージョン管理キー

### 3.2 固定費データ

```json
{
  "id": "uuid",
  "itemName": "家賃",
  "amount": 80000
}
```

### 3.3 支出データ

```json
{
  "id": "uuid",
  "amount": 1200,
  "itemName": "食材",
  "date": "2030-04-02",
  "category": "内食",
  "paymentMethod": "カード"
}
```

### 3.4 クラウド同期スナップショット

```json
{
  "version": 1,
  "updatedAt": "2026-04-03T00:00:00.000Z",
  "fixedCosts": [],
  "expenses": [],
  "categories": [],
  "paymentMethods": []
}
```

## 4. 画面仕様

### 4.1 ホーム（ダッシュボード）

- 表示内容
  - 選択月
  - 月次総額（固定費 + 選択月支出）
  - 支出件数
  - 支出一覧（新しい日付順、最大12件表示）
- 操作
  - 月移動（前月 / 翌月）
  - 行クリックで支出編集画面へ遷移
  - 各行の「削除」ボタンで該当支出を削除

### 4.2 支出入力

- 必須項目: 金額 / 項目名 / 日付 / カテゴリ / 支払方法
- カテゴリ・支払方法はその場で追加可能
- 編集時はタイトルが「編集」に切り替わる

### 4.3 固定費入力

- 固定費一覧表示
- モーダルから新規追加
- 各固定費に削除ボタンあり

### 4.4 分析

- 対象は選択月の「支出」（固定費は除外）
- 折れ線グラフ
  - 当月の累計支出
  - 過去月同日の累計平均
- 円グラフ
  - 支払方法別の割合と金額

### 4.5 初期表示ルート

- 固定費が 0 件なら固定費画面を初期表示
- 1 件以上ならホーム画面を初期表示

## 5. Google Drive 同期仕様

### 5.1 設定箇所

`script.js` の `GOOGLE_CONFIG.CLIENT_ID` を設定してください。

### 5.2 利用スコープ

- `https://www.googleapis.com/auth/drive.file`

### 5.3 認証と同期

- 起動時に自動認証（サイレント）を試行
- 「Sync with Google」ボタンで対話認証
- ログイン後の挙動
  - Drive 上の `kakeibo_data.json` があれば読み込み
  - なければローカルデータをアップロード
- ローカルデータ更新時、ログイン済みなら自動で Drive に保存
- 「Logout」でトークン破棄

### 5.4 注意点

- 同期先ファイル名は `kakeibo_data.json`
- 同名ファイル検索で同期先を特定する実装です
- 同期状態はヘッダーで表示（Offline / Connecting... / Syncing... / Synced / Error）

## 6. ローカル起動手順

このアプリはビルド不要の静的ファイル構成です。

```bash
python3 -m http.server 8080
```

ブラウザで `http://localhost:8080` を開いてください。

## 7. 運用メモ

- ローカルデータ初期化
  - `localStorage` の `kakeibo_` プレフィックスキーを削除
- Google同期をやり直したい場合
  - Google Drive 上の `kakeibo_data.json` を削除して再ログイン
- アプリのバージョン値（`APP_VERSION`）が変更されると、起動時に `kakeibo_` キーのローカルデータと関連キャッシュがクリアされます

## 8. トラブルシューティング

### Q1. 同期ボタンが `Google unavailable` のまま

- ネットワークや Google API スクリプトの読み込み失敗の可能性があります
- ブラウザ開発者ツールの Console ログを確認してください

### Q2. 同期できない / `Error` 表示になる

- `CLIENT_ID` 設定値の誤りを確認
- OAuth クライアントの JavaScript origin（例: `http://localhost:8080`）設定を確認

### Q3. 支出を編集したら並び順が変わった

- 編集は「該当IDを削除して新規追加」に近い保存処理です
- 一覧は日付降順で再描画されるため、日付変更時に表示位置が変わります

### Q4. データが見えない

- 表示中の月が意図通りか確認
- 別ブラウザ/別端末では `localStorage` は共有されません
