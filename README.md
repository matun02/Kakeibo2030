# Kakeibo2030

2030年の家計管理を想定した、シンプルなブラウザ完結型の家計簿アプリです。
固定費と日々の支出を分けて入力し、月単位の合計確認と簡易分析（日次推移・支払方法比率）を行えます。

## 1. アプリ概要

- **実行環境**: モダンブラウザ（HTML/CSS/Vanilla JavaScript）
- **データ保存先**: `localStorage` + Google Drive `appDataFolder`（任意同期）
- **画面構成**:
  - ホーム（ダッシュボード）
  - 支出入力
  - 固定費入力
  - 分析（折れ線グラフ・円グラフ）
- **主なユースケース**:
  1. 固定費を登録
  2. 日々の支出を入力
  3. 月を切り替えて総額・明細を確認
  4. 分析画面で傾向を確認

---

## 2. 技術構成

- `index.html`: 画面骨格（各セクション、フォーム、モーダル、タブ）
- `styles.css`: レイアウト・配色・レスポンシブスタイル
- `app.js`: 状態管理、永続化、描画、イベント制御
- `GoogleDriveService.js`: Google Identity Services + Drive API v3 による同期処理

Canvas API でグラフを描画し、Google同期時のみ以下を読み込みます。

- Google API Client (`https://apis.google.com/js/api.js`)
- Google Identity Services (`https://accounts.google.com/gsi/client`)

---

## 3. データ設計

### 3.1 localStorageキー

- `kakeibo_fixed_costs`: 固定費配列
- `kakeibo_expenses`: 支出配列

### 3.2 固定費データモデル

```json
{
  "id": "uuid",
  "itemName": "家賃",
  "amount": 80000
}
```

### 3.3 支出データモデル

```json
{
  "id": "uuid",
  "amount": 1200,
  "itemName": "食材",
  "date": "2030-04-02",
  "category": "内食",
  "paymentMethod": "カード"
}
```

### 3.4 集計仕様

- **固定費合計**: 登録済み固定費の全件合算（毎月同額として扱う）
- **選択月支出合計**: `date` が選択年月と一致する支出のみ合算
- **月次表示総額**: `固定費合計 + 選択月支出合計`

## 4. 画面設計

## 4.1 ホーム（ダッシュボード）

- 表示要素
  - 選択月ナビゲーション（前月/翌月）
  - 月次総額
  - 内訳（固定費 + 選択月支出）
  - 選択月の支出一覧（最大12件を新しい日付順）
  - 支出件数
- 操作
  - 月切替ボタンで `selectedMonth` を変更し再描画
  - 支出削除ボタンで対象データを永続層から削除
  - 「支出入力へ」「固定費入力へ」で画面遷移

## 4.2 支出入力

- 入力項目
  - 金額（数値、必須、最小1）
  - 項目名（文字列、必須）
  - 日付（必須、初期値は当日）
  - カテゴリ（内食/外食/生活用品/趣味/交際費）
  - 支払方法（現金/カード/PayPay/楽天ペイ/楽天キャッシュ）
- 保存時処理
  - UUID採番
  - 数値変換・トリム
  - localStorageへ追加保存
  - フォームリセット後ホームへ戻る

## 4.3 固定費入力

- 一覧表示（項目名・金額・削除）
- 「新規追加」でモーダルを開く
- モーダル項目
  - 項目名（必須）
  - 金額（数値、必須、最小1）
- 追加時処理
  - UUID採番
  - localStorageへ追加保存
  - モーダルを閉じて一覧再描画

## 4.4 分析

- 対象データは**選択月の支出のみ**（固定費は除外）
- 折れ線グラフ（日次推移）
  - 当月の日数分の配列を作成
  - 日単位に支出を加算
  - 最大値を基準にスケーリングして描画
- 円グラフ（支払方法比率）
  - 支払方法ごとに金額集計
  - 合計に対する割合を角度化して扇形描画
  - 凡例に割合（%）と金額を表示
- 対象データが無い場合は空メッセージを表示

## 5. 画面遷移/状態管理

### 5.1 主要状態

- `selectedMonth`: 表示対象月（`Date`）
- 表示中スクリーン: `dashboard | expense | fixed | analysis`

### 5.2 初期ルーティング

- 固定費が0件の場合: 固定費入力画面を初期表示
- 1件以上ある場合: ホーム画面を初期表示

### 5.3 再描画方針

- `renderAll()` を起点に、表示中画面に応じて必要な描画のみ実行
- 月変更・追加・削除後に再描画してUI整合性を維持

---

## 6. バリデーション/安全性

- HTMLの `required` / `min` による基本バリデーション
- `itemName` は `trim()` で前後空白を除去
- 支出一覧表示時は `escapeHtml()` でHTMLエスケープ（XSS対策）
- localStorage読み込み失敗時は空配列にフォールバック

## 9. Google Drive同期の設定

### 9.1 設定値（公開リポジトリ対応）

`app.js` の `GOOGLE_CONFIG` にプレースホルダーがあります。

- `CLIENT_ID`: OAuth 2.0 Client ID
- `API_KEY`: Browser API key

### 9.2 OAuth同意画面/認証設定

静的サイト（例: GitHub Pages）で利用する場合、Google Cloud Consoleで以下を許可してください。

- JavaScript origins: `https://<username>.github.io`
- （必要なら）localhost開発用 origin: `http://localhost:<port>`

本アプリはGISのトークンフローを使い、`drive.appdata` スコープで `appDataFolder` の `kakeibo_data.json` を読み書きします。

### 9.3 同期仕様

- ログイン直後に `appDataFolder` を確認
  - `kakeibo_data.json` が存在する場合: クラウドデータでローカルを上書き
  - 存在しない場合: ローカルデータを初期アップロード
- 支出/固定費の追加・削除ごとにローカル保存後、バックグラウンド同期
- ヘッダに同期状態（Offline / Syncing... / Synced / Sync error）を表示

## 10. ローカルでの起動方法

このアプリはビルド不要の静的サイトです。以下のいずれかの方法で `index.html` を配信してください。

### 10.1 最小手順（Python）

```bash
python3 -m http.server 8080
```

ブラウザで `http://localhost:8080` を開くと動作確認できます。

### 10.2 VS Code Live Server

1. `index.html` を開く
2. **Go Live** を実行
3. 自動で開いたローカルURLで動作確認

> 補足: `file://` で直接開いても利用できますが、将来的なAPI連携や同一オリジン挙動を考慮するとローカルサーバー経由を推奨します。

## 11. 運用メモ

- **データ削除（ローカル）**  
  ブラウザの開発者ツールから `localStorage` の以下キーを削除します。
  - `kakeibo_fixed_costs`
  - `kakeibo_expenses`

- **Google同期の再初期化**  
  `appDataFolder` 上の `kakeibo_data.json` を削除すると、次回ログイン時にローカルデータが再アップロードされます。

- **バックアップ推奨**  
  長期利用時は、定期的にエクスポート機能（今後実装予定）またはDrive側データ保全を行ってください。
