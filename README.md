# Kakeibo2030（ペイヤン）

ブラウザだけで動く家計簿アプリです。固定費・日々の支出を記録し、月次集計と簡易分析（累計推移・カテゴリ比率・支払方法比率）を確認できます。Google Drive 同期にも対応しています。

## 1. 現在の機能

- 固定費の登録 / 削除（**月ごと**に管理）
- 固定費の「この月を設定済みにする」運用
- 支出の登録 / 削除 / 編集（一覧行クリック）
- 支出の再登録（同じ項目を当日付で複製）
- 月切り替え表示（前月 / 翌月）
- 月次合計表示（当月固定費 + 当月流動費）
- 分析画面
  - 当月「累計支出」ライン
  - 過去月同日の「累計平均」ライン
  - カテゴリの円グラフ
  - 支払方法の円グラフ
- カテゴリ・支払方法の追加（入力画面からカスタム追加可能）
  - 使用頻度順で選択肢を並び替え
- `localStorage` への保存
- Google Drive への手動ログイン同期 / 起動時自動認証試行

## 2. 技術構成

- `index.html`
  - 4画面（ホーム / 支出入力 / 固定費入力 / 分析）
  - Google同期ボタン
  - 下部タブナビゲーション
- `styles.css`
  - モバイル寄りの単一カラムUI
  - カードレイアウト、モーダル、下部タブなど

  - 画面制御、状態管理、描画、イベント処理、ストレージ入出力

  - Google Identity Services と Drive API の認証/読み書き

> 補足: `index.html` から読み込まれる実行ファイルは `script.js` です。

## 3. データ構造と保存先

### 3.1 localStorage キー

- `kakeibo_fixed_costs`: 固定費配列
- `kakeibo_fixed_cost_month_status`: 月ごとの固定費設定済みフラグ
- `kakeibo_fixed_cost_review_prompted_at`: 固定費見直しプロンプト表示履歴
- `kakeibo_expenses`: 支出配列
- `kakeibo_categories`: 追加カテゴリ配列
- `kakeibo_payment_methods`: 追加支払方法配列
- `kakeibo_app_version`: アプリバージョン管理キー

### 3.2 固定費データ

```json
{
  "id": "uuid",
  "monthKey": "2030-04",
  "itemName": "家賃",
  "amount": 80000
}
```

### 3.3 支出データ

```json
{
  "id": "uuid",
  "amount": 1200,
  "itemName": "食材",
  "date": "2030-04-02",
  "category": "内食",
  "paymentMethod": "カード"
}
```

### 3.4 クラウド同期スナップショット

```json
{
  "version": 1,
  "updatedAt": "2026-04-03T00:00:00.000Z",
  "fixedCosts": [],
  "fixedCostMonthStatus": {},
  "fixedCostReviewPromptedAt": "",
  "expenses": [],
  "categories": [],
  "paymentMethods": []
}
```

## 4. 画面仕様

### 4.1 ホーム（ダッシュボード）

- 表示内容
  - 選択月
  - 月次総額（当月固定費 + 当月支出）
  - 固定費の設定状態（設定済み / 未設定）
  - 支出件数
  - 支出一覧（新しい日付順、最大12件表示）
- 操作
  - 月移動（前月 / 翌月）
  - 行クリックで支出編集画面へ遷移
  - 「再登録」ボタンで当日付に複製
  - 「削除」ボタンで支出削除

### 4.2 支出入力

- 必須項目: 金額 / 項目名 / 日付 / カテゴリ / 支払方法
- カテゴリ・支払方法はその場で追加可能
- 編集時はタイトルが「編集」に切り替わる

### 4.3 固定費入力

- 固定費は選択月単位で表示
- モーダルから新規追加
- 各固定費に削除ボタンあり
- 「この月を設定済みにする」で月次確定

### 4.4 分析

- 対象は選択月の「支出」（固定費は除外）
- 折れ線グラフ
  - 当月の累計支出
  - 過去月同日の累計平均
- 円グラフ
  - カテゴリ別比率
  - 支払方法別比率

## 5. Google Drive 同期仕様

### 5.1 設定箇所

`script.js` の `GOOGLE_CONFIG.CLIENT_ID` を設定してください。

### 5.2 利用スコープ

- `https://www.googleapis.com/auth/drive.file`

### 5.3 認証と同期

- 起動時に自動認証（サイレント）を試行
- 「Sync with Google」ボタンで対話認証
- ログイン後の挙動
  - Drive 上の `kakeibo_data.json` があれば読み込み
  - なければローカルデータをアップロード
- ローカルデータ更新時、ログイン済みなら Drive に保存
- 「Logout」でトークン破棄

### 5.4 注意点

- 同期先ファイル名は `kakeibo_data.json`
- 同期状態はヘッダーで表示（Offline / Connecting... / Syncing... / Synced / Error）

## 6. ローカル起動手順

このアプリはビルド不要の静的ファイル構成です。

```bash
python3 -m http.server 8080
```

ブラウザで `http://localhost:8080` を開いてください。

---

## 7. 運用メモ

- ローカルデータ初期化
  - `localStorage` の `kakeibo_` プレフィックスキーを削除
- Google同期をやり直したい場合
  - Google Drive 上の `kakeibo_data.json` を削除して再ログイン
- アプリのバージョン値（`APP_VERSION`）が変更されると、起動時に `kakeibo_` キーのローカルデータと関連キャッシュがクリアされます
